# Безопасность

[English](SECURITY.md) | [Русский](SECURITY.ru.md)

RepoForge предназначен для создания чистых архивов проектов с исключением распространённых build-артефактов, локальных файлов, ключей подписи и секретов.

## Исключаемые чувствительные файлы

RepoForge исключает распространённые чувствительные файлы:

- `local.properties`
- `.env`
- `.env.local`
- `secrets.properties`
- `google-services.json`
- `service-account.json`
- `*.jks`
- `*.keystore`
- `*.pem`
- `*.key`
- `*.p12`
- `*.pfx`
- `*.crt`
- `*.cer`

## Важное замечание

RepoForge снижает риск случайной отправки чувствительных файлов, но пользователь всё равно должен проверять созданные архивы перед публикацией или отправкой третьим лицам.

## Сообщение о проблемах

Проблемы безопасности можно сообщать через GitHub Issues.
